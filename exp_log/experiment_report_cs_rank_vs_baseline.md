# 实验报告：横截面标签处理与原始绝对收益基线的对比研究

## 1. 核心摘要 (Executive Summary)

针对近期系统重构后多因子量化选股模型的表现，我们设计并执行了**单变量对比实验**。本实验重点探究了训练目标（Label）的**横截面处理方式**对 LightGBM 模型预测能力与最终投资组合表现的影响。

**核心结论**：
1. **「信号削弱假说」被强力证实**：启用横截面 rank 标准化 (`cs_rank`) 会使模型的平均 Rank IC 从 **0.0188** 骤降至 **0.0040**，多空组合年化夏普比率（基于日多空）从 **0.1643** 萎缩至 **0.0135**。这确凿地证明了横截面 rank 变换在此特征集上极大地削弱了有效预测信号。
2. **重构特征库正确性验证**：在禁用标签处理器的情况下，基线模型成功复现并超越了 `README.md` 中记录的 `Rank IC ≈ 0.015` 基准线，证明新版 8 因子动态提取与重构模块（`kbar.py` 等）计算逻辑无误。
3. **发现并修复评估隐患**：在对比过程中，我们发现并定位了 `model/evaluate.py` 分组收益评估中的一个严重 Bug：当模型树数较少（早停于 7~11 棵）导致预测值高度离散时，`pd.qcut` 的同值合并机制会导致部分交易日的分组数不足 5 组，从而在多空序列计算中被**静默丢弃**（基线有效天数从 486 降至 46 天）。我们通过引入 `s.rank(method="first")` 破同分机制彻底解决了这一隐患，实现了 486 天全样本的严谨、公平评测。

## 4. 评估 Bug 发现与破同分机制修复 (Technical Audit & Fix)

### 4.1 隐患成因：
在旧版的 `evaluate_quantile_returns` 中，分组逻辑如下：
```python
def _grp_of_day(s: pd.Series) -> pd.Series:
    if len(s) < n_groups * 4:
        return pd.Series(float("nan"), index=s.index)
    return pd.qcut(s, n_groups, labels=False, duplicates="drop")
```
由于当前配置中 LightGBM 树数极少（在 10 棵树左右），导致模型在测试集上预测出来的预测值序列极其离散（包含大量重复值）。
在 `pd.qcut` 中，若分位数边界因重复值过多而重合，`duplicates="drop"` 参数会**自动合并/丢弃多余的区间**。这导致：
* 许多交易日的返回值只包含 `[0, 1, 2]` 或 `[0, 1, 2, 3]` 等不足 5 个组别。
* 随后的 `unstack("grp")` 将无法在该交易日生成 Group 4.0 的数据。
* 计算 `daily_ls = (daily_grp.iloc[:, -1] - daily_grp.iloc[:, 0]).dropna()` 时，任何缺失 Group 4.0 的交易日都会被 `dropna()` **彻底滤除**。
* **后果**：基线测试中 486 个有效天数被默默过滤得只剩下 46 天，扭曲了组合评价的真实图景。

### 4.2 破同分机制修复方案：
我们在 `model/evaluate.py` 中引入 `s.rank(method="first")` 对预测值进行预排序破同分，保证了每个股票的排序值唯一，从而让 `pd.qcut` 能够将股票百分之百完美均分至 5 个组中，消除了所有 NaN 的生成：
```diff
     def _grp_of_day(s: pd.Series) -> pd.Series:
         # s = 某一天所有股票 of pred。当天股票太少则不分组(返回 NaN)
         if len(s) < n_groups * 4:
             return pd.Series(float("nan"), index=s.index)
-        # 把当天股票按 pred 从低到高切成 n_groups 等份,组号 0..n_groups-1
-        return pd.qcut(s, n_groups, labels=False, duplicates="drop")
+        # 用 rank(method="first") 破同分，确保均分且不因同值预测导致 bin 合并丢失交易日
+        return pd.qcut(s.rank(method="first"), n_groups, labels=False)
```

## 5. 理论探讨：为什么横截面标签处理削弱了信号？ (Theoretical Discussion)

1. **绝对幅度与非线性波动的丧失**：
   `cs_rank` 强行将每天的收益率映射到 $[-1.73, 1.73]$ 的均匀对称区间，完全剥离了日度波动幅度的差异（例如，大牛市日单日涨幅 $10\%$ 与盘整日单日涨幅 $1\%$ 被赋予相同的 Rank 值）。
   我们的特征集中包含了大量描述绝对波动率与极端动量的特征（如 `STD5`, `STD20`, `STD60`, `KLEN`）。这些特征天生与收益率的**绝对规模（Scale）**正相关。将标签标准化后，这种“尺度对应”的强逻辑链条被强行斩断，导致模型无法学到规模相关的规律。

2. **CSZScoreNorm 与 CSRankNorm 的对比**：
   相比 `cs_rank`（Rank IC = 0.0040），`cs_zscore`（Rank IC = 0.0086）表现稍好，这正是因为 Z-Score 仅做了线性的平移与缩放，**保留了日内股票间收益差额的绝对基数（Cardinal Distances）**，使得极端涨幅的样本依然能获得极高的正向 Z-Score。这再次佐证了“保留绝对相对差距”对非线性树模型的重要性。

3. **测试集与训练集的目标错配**：
   由于在推理侧（测试集）评估时，我们使用的是真实的绝对涨跌幅（为了保证组合回测的绝对真实性），若在训练侧强行拟合横截面相对值，会导致“训练目标是相对排位，测试目标是绝对收益”的**错配**。在特征信号偏弱的金融场景下，这种错配会被无限放大。

## 6. 后续研究与配置建议 (Actionable Recommendations)

1. **保存最优配置**：
   我们已在 `config.yaml` 中默认**保持注释/禁用** `label_processors`。在当前 8 因子体系下，直接对绝对收益目标进行拟合是最优策略。

2. **下一步实验方向（引入真正的截面特征）**：
   如果未来仍然希望防范潜在的跨期泄漏或大盘 beta 污染，正确的做法**不是强行对 Label 做截面处理**，而是：
   * 在特征侧加入**截面特征**（如在特征计算中引入行业中性化或截面 rank 特征，如 `RANK_ROC20` 等）。
   * 保持 Label 依然是绝对收益不变，让树模型自动在绝对目标与截面特征之间寻找最优的非线性组合。
